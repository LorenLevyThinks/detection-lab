# detection-lab

Detection engineering lab focused on building structured, testable, and validated security detections across multiple SIEM platforms.

This repository demonstrates detection-as-code principles including:
- Structured rule identification
- MITRE ATT&CK mapping
- Platform translations (Sigma → Sentinel KQL → Splunk SPL)
- Test case artifacts
- Tuning guidance
- CI-based validation enforcement

# Depending on the nature of the environment these principles can be adjusted. 


## Philosophy

Detections are software.

They require:
- Version control
- Quality gates
- Testing artifacts
- Consistent structure
- Platform portability

This lab models detection lifecycle discipline similar to production environments.

# Lastly I beleive that tools such as the "Universal Translator" can significantly reduce time and enable accuracy for solving the problems of today.
